Data Structures and Algorithms

  1. Problems in Nature
    • Iteration: Solving problems using loops, real-time examples: Climate change, School schedules.
    • Recursion: Breaking problems into smaller subproblems, examples: Ants following food, Weeds in a garden.
    • Backtracking: Exploring all solutions and rejecting those that fail constraints, examples: Bees searching for flowers (pollination), Maze solving.
  2. Space and Time Efficiency
    • Space Efficiency: How quickly tasks can be completed using an algorithm.
    • Time Efficiency: Memory requirements for completing tasks.
    • Order of Growth:
      • O(1) - Constant, example: accessing one array index value.
      • O(n) - Linear, example: traversing an array.
      • O(n^3) - Cubic.
      • O(log n) - Logarithmic, example: Binary search.
      • O(n^2) - Quadratic, example: Bubble sort.
      • O(n log n) - Linearithmic.
      • O(n^k) - Polynomial.
      • O(2^n) - Exponential.
      • O(n!) - Factorial.
  3. Design Principles
    • Divide and Conquer: Breaking data into smaller parts, solving, and combining results, example: Merge sort.
  4. Tree Data Structures
    • BST: Binary Search Tree with faster operations.
    • AVL: Self-balancing BST using rotations.
    • 2-3 Tree: BST with nodes having two or three children.
    • Red-Black Tree: Balanced binary tree with specific properties.
    • Heap: Priority data tree, example: Priority queues.
    • Trie: Tree structure for fast prefix-based lookups.
  5. Array Query Algorithms
    • Applications: Quickly calculate sum of elements in a range.
    • Fenwick Trees: Efficient cumulative frequency or sum calculation.
  6. Difference Between Tree and Graph Traversals
    • Tree: Hierarchical, acyclic, n-1 edges for n nodes.
    • Graph: General structure, cyclic, up to n(n-1)/2 edges.
  7. Sorting Algorithms
    • Bubble Sort: Adjacent comparisons, O(n^2).
    • Selection Sort: Finds smallest element, O(n^2).
    • Insertion Sort: Inserts element in sorted part, O(n^2).
    • Quick Sort: Uses pivot, O(n log n).
    • Merge Sort: Divide and merge, O(n log n).
    • Heap Sort: Builds heap tree, O(n log n).
  8. Shortest Path Algorithms
    • Dijkstra: Shortest path with positive weights.
    • Floyd: All-pairs shortest path using dynamic programming.
    • Bellman-Ford: Handles negative weights.
    • Warshall: Uses bitwise operations.
    • Kruskal: Minimum spanning tree by edges.
    • Prim: Minimum spanning tree starting from any node.
  9. Algorithm Design Techniques
    • Divide and Conquer: Break, solve, and combine.
    • Brute Force: Try every solution.
    • Backtracking: Explore step-by-step, retry failures.
    • Recursion: Solve smaller versions of the problem.
